# Wellness Portal 2.4 — печат, навигация и пълен backup/restore

## Промени

### Печат без празна начална област
- Печатът на „История по дати“ и „Структура“ вече изолира само полезната секция.
- Банерът, менюто, празните контейнери и останалата част от страницата не участват в печатния layout.
- Печатното съдържание започва от горния край на първата страница.

### Видим бутон „Назад“
- Бутонът е преместен от падащото меню над основното съдържание.
- Показва се в профил, измервания, акаунти, структура, настройки и backup/restore.
- При клиентски акаунт връща към неговия начален профил; при треньор/администратор измерванията връщат към клиентския профил.

### Администраторски Backup и Restore
- Добавена е отделна опция „Backup и Restore“, видима само за администратор.
- Backup файлът включва:
  - Supabase Auth акаунтите и достъпните им метаданни;
  - профили, роли и йерархия;
  - клиентски записи;
  - параметри;
  - всички измервания;
  - снимките от `client-photos`.
- Restore заменя приложните данни с архива и възстановява връзките между акаунти, клиенти и йерархия.
- Съществуващите Auth акаунти се разпознават по ID или имейл и запазват текущите си пароли.
- Ако акаунт липсва в проекта, той се създава с временна парола, която администраторът може да изтегли след restore.

> Ограничение: Supabase Admin API не предоставя паролните хешове. За пълен инфраструктурен архив на самия Supabase проект използвай и официалния Database Backup/CLI backup.

### Мерки на тялото
- Датата вече не се повтаря до всяка последна стойност.
- Показва се в заглавието „Последна стойност“, както при Tanita измерванията.

### Клиентски акаунт
- В „История по дати“ са премахнати бутоните „Импорт“ и „Експорт“ за клиентски акаунти.
- Прегледът, печатът и разрешеното редактиране на мерки остават налични.

## Обновяване

SQL миграция не е необходима за версия 2.4.

Публикувай отново Edge Function-а:

```bash
supabase functions deploy manage-account --no-verify-jwt
```

След това build-ни и публикувай frontend-а:

```bash
npm install
npm run build
```
