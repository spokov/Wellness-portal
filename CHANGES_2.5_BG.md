# Wellness Portal 2.5 — банер, двуезични параметри и флагове

## Промени

### Банер по цялата ширина
- Премахнато е ограничението от 1200 px.
- Банерът вече заема цялата ширина на екрана на desktop, tablet и mobile.
- Запазено е адаптивното изрязване на малки екрани.

### Български и английски имена на параметрите
- Таблицата `parameters` вече пази отделно `name` (български) и `name_en` (английски).
- При английски интерфейс параметрите се показват с английското си име във:
  - Tanita измервания;
  - мерки на тялото;
  - последна стойност;
  - история по дати;
  - CSV export;
  - съобщенията за валидация.
- Стандартните 18 параметъра получават готов английски превод чрез `migration_12.sql`.
- В администраторските настройки има две отделни полета за всяко име.
- Нов параметър изисква българско и английско име.
- CSV import разпознава и двете имена, което запазва съвместимостта със стари файлове.
- Backup/restore включва `name_en`; стар backup без това поле се възстановява с fallback към българското име.

### Смяна на езика с флагове
- Бутоните `BG` и `EN` са заменени с видими 🇧🇬 и 🇬🇧 флагове.
- На по-широк екран се показват и съкращенията BG/EN.
- Активният език остава ясно маркиран и бутоните имат достъпни `aria-label` и `title` атрибути.

## Обновяване

1. Направете backup на Supabase базата.
2. Изпълнете в SQL Editor:

```text
supabase/migration_12.sql
```

3. Публикувайте актуалната Edge Function:

```bash
supabase functions deploy manage-account --no-verify-jwt
```

4. Build и deploy на frontend-а:

```bash
npm install
npm run build
```

`migration_12.sql` не изтрива параметри или измервания.
