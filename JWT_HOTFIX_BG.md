# Поправка за `invalid JWT ... ES256 ... kid <nil>`

## Причина

Supabase Edge Functions имат предварителна JWT проверка, която се изпълнява преди
`manage-account`. При някои конфигурации след преминаване към асиметрични ES256
ключове или нов publishable API key заявката се отхвърля от този слой.

`manage-account` вече извършва собствена задължителна проверка на потребителя чрез
`supabase.auth.getUser()` и след това проверява ролята му в `profiles`. Затова
предварителната проверка може безопасно да бъде изключена само за тази функция.

## Публикуване

От основната папка на проекта изпълнете:

```bash
supabase login
supabase link --project-ref YOUR_PROJECT_REF
supabase functions deploy manage-account --no-verify-jwt
```

Файлът `supabase/config.toml` също съдържа:

```toml
[functions.manage-account]
verify_jwt = false
```

След публикуването излезте от портала и влезте отново, за да се издаде нов
access token. След това опитайте отново да добавите клиент.

## Сигурност

Функцията не става публично достъпна в практически смисъл: всяка заявка без
валиден потребителски access token се отхвърля от `auth.getUser()`, а действията
за управление на акаунти се разрешават само на профили с роля `admin` или
`trainer` и според йерархичните проверки в самата функция.
